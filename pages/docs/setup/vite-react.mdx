# React with Vite Setup
### steps

import { Tabs } from "nextra/components";

1. Open your terminal.
2. Run the following command to create a new React app:

<Tabs items={["npm", "yarn", "pnpm", "bun"]}>
  <Tabs.Tab>
  ```bash copy
  npm create vite@latest
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash copy
  yarn create vite
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash copy
  pnpm create vite
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash copy
  bun create vite
  ```
  </Tabs.Tab>
</Tabs>

This will initiate the creation of a new Vite-powered React project.

3. Follow the prompts to enter your project name and select the desired template. 
4. Choose `React` and `javascript` when prompted.
5. Navigate to Project Directory:
```bash copy
  cd your-project-name
```
6. Install Dependencies.

<Tabs items={["npm", "yarn", "pnpm", "bun"]}>
  <Tabs.Tab>
  ```bash copy
  npm install
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash copy
  yarn install
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash copy
  pnpm install
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```bash copy
  bun install
  ```
  </Tabs.Tab>
</Tabs>

7. Copy the following CSS code into your `index.css` file to apply **global variables** to your project

```css copy
@import url("https://fonts.googleapis.com/css2?family=Poppins:ital,wght@0,100;0,200;0,300;0,400;0,500;0,600;0,700;0,800;0,900;1,100;1,200;1,300;1,400;1,500;1,600;1,700;1,800;1,900&display=swap");

:root {
  --primary: 200 100% 50%;
  --primary-forground: 218 100% 50%;
  --secondary: 319 100% 50%;
  --secondary-forground: 277 100% 50%;
  --text: 255 100% 0%;
  --text-forground: 255 100% 100%;
  --sub-text: 255 0% 30%;
  font-family: "Poppins", Tahoma, Geneva, Verdana, sans-serif;
}

.dark {
  --primary: 200 100% 50%;
  --primary-forground: 218 100% 50%;
  --secondary: 319 100% 50%;
  --secondary-forground: 277 100% 50%;
  --text: 255 100% 100%;
  --text-forground: 255 100% 0%;
  --sub-text: 255 0% 30%;
}
```
This code defines the global color variables and imports the 'Poppins' font for consistent styling across your application.

8. Create a folder named `sectioner` within the `src/components` directory. Organize your components into separate folders within `sectioner` for better project structure and maintainability.

**note**: It's recommended to create a separate folder for each section or component within the `sectioner` directory.
